context

For young adults who want to make online polls to help make faster decisions, pollit is a mobile app that allows users to create, vote on, and view polls. During Spring 2022, I started doing the UI design and frontend development for pollit with my two friends from Brown University, Kaan Ozulkulu (backend) and Daniel Civita (backend).

app breakdown
Pollit allows users to do three things: create polls, vote on polls, and view polls on their feed. Think Twitter, but with polls instead of tweets. Users can create public or private polls. Public polls mean that any pollit user can vote. Private polls means that only friends of the poll creator can vote. In the future, we plan to take this a step further by allowing poll creators to choose which specific users are allowed to vote. For example, if you're planning a dinner party with 10 friends, you can ensure that only those 10 friends can vote on food preferences.

If the results of a poll are time-sensitive, the creator set the poll's duration i.e. for how long users will be able to vote before the poll is closed.

We decided to build a UI which combines aspects from Instagram, TikTok, and BeReal, because we felt these design choices were intuitive.
technology used
  • Frontend: React Native
  • Backend: Python
  • Database: PostgreSQL
page breakdown
Below is a breakdown of each page of the app, with a list of the features on that page.
  • Login + Create Account page: username and password authentication. This also includes a link to a Forgot Password page
  • Feed page: allows users to scroll through polls. The feed is split into 3 sections:
    • My Friends: Allows user to scroll through polls created by their friends.
    • Trending: Allows users to scroll through popular public polls.
    • My Votes: Allows users to scroll through the polls they have already voted on.
    Each poll on the feed also includes:
    • The poll's question
    • The creator of the poll
    • The date the poll was created
    • Time left before the poll expires
    • A clickable component for option 1 and option 2. Once the user votes, they can see the percentage of people who voted for each option.
    • An icon representing whether the poll is private or public
    • A menu icon which, when clicked, allows user to edit or delete the poll if they're the creator, or report the poll if they're not the creator.
  • Poll details: If a user clicks on a poll on the feed, it navigates them to a page with that specific poll's details, specifically the names of the users that voted for each option. Eventually this will also include a "Share Results" button that will produce a link so that users can send a poll's results to friends through text.
  • User's profile: Includes profile picture, the number of friends that user has, and a navigation bar with two categories: (1) all of the polls the user created and (2) all of the polls the user has voted on
  • Create poll page: Allows user to create a poll. This includes:
    • Inputting a question
    • Inputting choice 1 and choice 2
    • Choosing whether the poll should be public or private from a dropdown menu
    • Setting the poll duration from a dropdown menu (1 hour, 1 day, 1 week, or no limit)
    • Cancel button which navigates user back to feed
    • "Create Poll" button which posts the poll to the backend and navigates user back to feed
  • Friends page: Allows user to search for users by username, accept or decline friend requests, and view all of their existing friends.
design choices + accessibility
  • We decided to create a mobile app as opposed to a web app. We thought this would be more practical since users will most likely think of questions to poll throughout the day, when they're not necessarily in front of their computer. This would also enable users to easily and quickly vote on polls from their phone.
  • Mimicking popular apps makes it easier for users to navigate our app.
  • In terms of accessibility, we did the following:
    • good color contrast
    • optimal button sizes
    • consistency across the app
    • bottom navigation bar allows users to easily "undo" actions
    • captions for images can be read by screen readers for visually impaired users
security + privacy
  • Privacy
    • We give users the choice between making polls public or private.
    • We will require users to agree to our privacy policy when signing up. This is because we plan to launch our app on the App Store, which requires a privacy policy if the app collects user data.
  • Security
    • We decided to use a PostgreSQL database because it is an existing API and is certified under major privacy and security standards.
    • Data Validation:
      • Email and password authentication
      • Checking length of user input to prevent system overload
      • Report button
